# ChronoTrace Phase 3: True Ingestion Pipeline 

Per your diagram, I will remove the `live_stream.py` mock generator entirely. We will pivot to a true **Data Pipeline Architecture** capable of ingesting diverse user-selected files (`.evtx`, `.pcap`, `.log`), normalizing them, running them through a structured detection engine, and pushing the alerts to the interactive dashboard.

## Overview of the Pipeline Architecture

1. **DATA SOURCES:** The frontend will be updated to allow users to select and upload multiple files of different types (`.pcap`, `.evtx`, `.csv`, `.log`).
2. **LOG INGESTION:** A Python router that detects the file extension and routes it to the specific parser.
3. **NORMALIZATION:** Standardizing the chaotic output into our `EventLog` schema with UTC timestamps.
4. **PROCESSING:** Merging the uploaded files and sorting them chronologically.
5. **DETECTION ENGINE:** Passing the sorted timeline through rule algorithms (Brute Force, Privilege Escalation).
6. **OUTPUT & VISUALIZATION:** Broadcasting the finalized, categorized events to the interactive UI.
